JSON和XML的比较

可读性

JSON和XML的可读性不相上下，JSON语法简易，XML是规范的标签形式

可扩展性

XML天生有很好的扩展性，JSON也有，没有什么是XML可以扩展而JSON却不能扩展的，不过JSON可以在js中存储js复合对象，有着xml不可比拟的优势

编码难度

JSON更容易，XML文档多很多结构上的字符

解码难度

XML很难，JSON中难度几乎为0

流行度

XML更流行，不过Ajax中都用JSON

数据体积

JSON体积小，传输速度更快，交互也更好

数据描述

XML对数据的描述更好

传输速度

JSON远快于XML

宏任务和微任务

运行机制

在事件循环中，每进行一次循环操作称为 tick，每一次 tick 的任务处理模型是比较复杂的，但关键步骤如下：

  • 执行一个宏任务（栈中没有就从事件队列中获取）
  • 执行过程中如果遇到微任务，就将它添加到微任务的任务队列中
  • 宏任务执行完毕后，立即执行当前微任务队列中的所有微任务（依次执行）
  • 当前宏任务执行完毕，开始检查渲染，然后GUI线程接管渲染
  • 渲染完毕后，JS线程继续接管，开始下一个宏任务（从事件队列中获取）

Promise中的then

首先看看官方文档中的定义

一个 promise 必须提供一个 then 方法以访问其当前值、终值和据因。

promise 的 then 方法接受两个参数：

promise.then(onFulfilled, onRejected)

如果

onFulfilled 和 onRejected 都是可选参数。

如果 onFulfilled 不是函数，其必须被忽略
如果 onFulfilled 是函数：

当 promise 执行结束后其必须被调用，其第一个参数为 promise 的终值
在 promise 执行结束前其不可被调用
其调用次数不可超过一次

then方法提供一个供自定义的回调函数，若传入非函数，则会忽略当前then方法。
回调函数中会把上一个then中返回的值当做参数值供当前then方法调用。
then方法执行完毕后需要返回一个新的值给下一个then调用（没有返回值默认使用undefined）。
每个then只可能使用前一个then的返回值。

dns解析

- 先在本地host查找

- 到浏览器dns缓存查找

- 到根域名服务器查找

- 找主域名服务器查找

- 返回ip地址

浏览器缓存

- 强缓存maxAge未过时直接读取本地磁盘缓存，无需建立连接

- 协商缓存if-modify-since –> etag （优先级高）

没有缓存，建立tcp连接

- 三次握手 syn -> syn+ack -> fin

- 建立SSL通信，客户端携带随机数，支持的加密算法列表请求443端口 -> 服务端返回数字证书+公钥+随机数b -> 客户端验证证书有效性生成随机数c，用某种加密算法根据abc生成对称密钥，并返回给服务端 -> 服务端私钥解密，得到对称密钥，然后双方使用对称密钥通信

// - 复杂请求，先发起option请求检验服务可用性

- 接收数据包（请求头，请求行，请求体）根据content-type进行解码，根据不同资源类型进行不同的加载策略

- 四次挥手 syn -> syn + ack -> fin -> ack + fin

浏览器渲染

- 浏览器主进程通知渲染进程可以开始渲染了

- 渲染进程接收到垂直同步信号，通知主线程

- 主线程先进行渲染的前置操作，清空当前的所有微任务，之后触发requestAnimationFrame，进入渲染阶段

- 构建dom树 + css renderer -> 得到layout树

-根据图层生成规则，并计算节点位置，生成layer树

- 通知合成线程，layer树准备完毕

- 合成线程进行分块和光栅化的操作

- 通知gpu可以开始进行绘制了

- gpu调用显示器提供的DirectX api，ldp，aldp不同策略进行得到色彩页面显示

- （中间会穿插css的加载，js的执行，defer，async等不同策略）
